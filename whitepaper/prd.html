Section 1
1. Product Vision & Problem Statement

Higher education institutions struggle with curriculum drift and manual audits. Lesson plans, lecture slides, and assignment briefs often drift from baseline syllabi or include outdated industry technologies, affecting student learning outcomes and accreditation compliance.

Core Objective: Provide a secure, automated, and server-side curriculum auditor that checks teaching materials against module learning outcomes and live web standards, yielding instant scores and recommendations.

Traditional Auditing

Manual review of module guides takes weeks, relies on peer subjective comments, and cannot easily verify the currency of all mentioned libraries/frameworks.

S.A.G.E. Solution

Instant recursive auditing using Fireworks AI GLM-5p2 (131k context) and Brave Search API to detect obsolete techniques and verify outcomes alignment.

Section 3
3. Recursive Material Ingestion Specs

Coordinators need a friction-free way to load course files. Rather than filling in tedious forms, S.A.G.E. mandates a structured local directory injection system:

Folder Path Syntax: repository/[term]/[faculty]/[lecturer]/[module_code]/[material_type]/[filename]

System Ingestion Logic:

  1. The backend employs a recursive folder scanner (built on PHP's RecursiveDirectoryIterator) to traverse the repository/ directory.
  2. It parses directories on-the-fly, extracting parameters (Term, Faculty, Lecturer, Module Code, and Document Type) from the path segments.
  3. It cross-checks files against existing records in audit_results table. If a file exists in the directory but has no database record, it is flagged as Untracked / Pending Audit.
  4. Allowed document types (material_type) are: lesson_plan, assignment, and lecture_material.
Section 4
4. AI-Powered Auditing Agent (Fireworks AI)

Once the Coordinator clicks Audit, the server initiates an autonomous XML-based agent loop utilizing AMD Fireworks AI.

Agent Specifications:

  • Inference Service: Exclusive Fireworks AI connection using the accounts/fireworks/models/glm-5p2 model.
  • Tool-Calling Loop: The agent issues structured commands using custom XML nodes. S.A.G.E. parses these outputs on the server, executes the tools, and feeds results back to the agent in subsequent messages.
  • Available Tools:
    • get_module_baseline: Returns database-defined Module Learning Outcomes (MLOs) and Assessment weighting for the module code.
    • web_audit: Invokes Brave Search to verify live tech status.
    • save_audit_result: Stores the final audit report, score, strengths, and recommendations JSON to MySQL.
Section 5
5. Brave Live Verification Tool (Web Audit)

To prevent hallucinations and evaluate the currency of technology, the agent must be able to search the live web.

Web Search Pipeline:

  1. The agent outputs: <call name="web_audit">query term</call>.
  2. The PHP backend intercepts this block, issues an authenticated GET request to the Brave Search API API (https://api.search.brave.com/res/v1/web/search).
  3. The top search result snippets are extracted, cleaned, and injected back to the agent context as a tool response: <response>[Live Web Info]</response>.
  4. The agent evaluates whether the textbook, framework version (e.g. Angular 2 vs. modern standards), or code snippet mentioned in the document matches current 2026 industry requirements.
Section 6
6. Core Database Schema Requirements

The relational schema must enforce referential integrity to connect users, modules, and logs:

  • users: Holds IDs, roles, and hashed credentials.
  • modules: Stores the source of truth for course details (Module Code, Credits, Faculty, Level).
  • module_learning_outcomes: Links multiple learning objectives to a single module code.
  • module_assessments: Dictates official assessment criteria (numbers, weights).
  • audit_results: Stores the finalized AI report outputs (score, weaknesses list, strengths, and suggestions) tied to the ingested file path.
  • chat_logs: Tracks user queries and AI replies in S.A.G.E. Chat.
Section 7
7. System Constraints & Non-Functional Requirements
Security (Server-Side Rule)

No external API keys (Fireworks AI, Brave Search) are ever loaded or sent to the client-side browser. All calls run through secure, PHP server cURL commands protected by server environment settings (.env).

Shared Infrastructure Performance

Operating under zero-budget constraints means the application must execute efficiently in memory-constrained shared PHP environments. Recursive iterators must release file pointers immediately after ingestion.

Audit Loop Resilience

The agent loop is limited to a maximum of 5 iterations per audit command to prevent infinite loops, API rate-limiting blocks, and thread execution timeout on the shared server.

Responsive Design System

Must support desktop, tablet, and mobile browsers using pure, responsive CSS layout. Fonts must render smoothly using Space Grotesk for headers and modern sans-serif for reading.
