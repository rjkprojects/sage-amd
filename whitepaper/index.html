S.A.G.E.
System for Academic Governance and Evaluation
An AI-powered academic audit and governance platform designed to ensure learning outcomes alignment, syllabus consistency, and curriculum relevance across higher education faculties. Powered exclusively by AMD Fireworks AI inference and Brave Search tooling.

S.A.G.E. has been rebuilt specifically as a hackathon project submission for the AMD Developer Hackathon: Act II. How it works
End-to-end governance in one flow
S.A.G.E. automatically cross-references uploaded teaching materials against the syllabus definitions stored in the system database. Let's trace how the academic review happens:
01
Authentication and Role Routing
User logs in to their dashboard. S.A.G.E. provides distinct custom portals for three academic roles: Lecturers, Coordinators, and the Academic Lead.
02
Materials Intake
Coordinators drop lesson plans, assignments, and lecture materials into the repository. Documents are parsed and registered under specific module directories.
03
Agent-driven Auditing
The coordinator triggers an audit. The backend calls the unified Fireworks AI model, executing an autonomous agent loop that calls system tools using XML syntax.
04
Brave Search Web Verification
If the agent encounters technology names or citations, it executes the Web Audit tool to check live data using Brave Search, detecting outdated material.
05
Results Saved & Visualized
The agent saves the finalized report containing scores, strengths, weaknesses, and improvement lists directly to MySQL for dashboards and feedback cards.
Core Engine
System Architecture
S.A.G.E. connects standard web technologies with an autonomous agent loop, keeping code operations server-side to guarantee API key security and high reliability.

Server-Side Processing Design: All API requests to Fireworks AI and Brave Search are routed through backend handlers (agent.php, chat.php, and nexus.php) rather than client-side scripts. This shields credentials, prevents cross-origin resource issues, and ensures security.

The core integration is unified around a single, robust inference function, callFireworks(), defined across all processing entry points:

function callFireworks($messages) { $ch = curl_init(FIREWORKS_API_URL); $payload = json_encode([ 'model' => FIREWORKS_MODEL, 'max_tokens' => 131072, 'top_k' => 40, 'presence_penalty' => 0, 'frequency_penalty' => 0, 'messages' => $messages ]); curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); curl_setopt($ch, CURLOPT_POST, true); curl_setopt($ch, CURLOPT_POSTFIELDS, $payload); curl_setopt($ch, CURLOPT_HTTPHEADER, [ 'Accept: application/json', 'Content-Type: application/json', 'Authorization: Bearer ' . FIREWORKS_API_KEY ]); ... }

0.0

Optimized with presence_penalty and frequency_penalty at 0, ensuring consistent structural JSON outputs for database storage.

Agentic Tooling
Brave Search Integration
A critical weakness of traditional curriculum audits is that static LLMs are unaware of modern trends. S.A.G.E. solves this by granting the AI agent a web audit tool powered by the Brave Search API.

During the audit, if the LLM needs to verify if a technology tool or framework is outdated, it emits a specific tool block. S.A.G.E. catches this call on the server side, queries Brave Search, and pushes the live search results back to the LLM to complete its evaluation.

Conclusion

S.A.G.E. does not replace the human touch of educators. Instead, it handles structural consistency and currency checks so academics can focus on teaching.
